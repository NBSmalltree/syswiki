# 系统百科平台 — 使用说明书

## 1. 系统简介

系统百科平台是面向银行内网的统一系统知识管理系统，旨在打破信息孤岛，将分散在各处的系统文档、技术架构、服务器配置、运维SQL等信息集中管理。平台支持多系统入驻，每个系统拥有独立的百科空间。

**核心功能：**
- 系统空间管理：多系统入驻，独立空间隔离
- 百科内容管理：Markdown驱动的在线编辑与离线导入导出
- 黄金链路拓扑：ECharts动态拓扑图，支持点击下钻查看接口详情
- 运维SQL库：参数化SQL模板，一键生成与复制
- AI智能问答：基于知识库的RAG问答，支持快速/深度两种模式

## 2. 环境准备与启动

### 2.1 环境要求

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 11+ | 推荐 OpenJDK 11 |
| Maven | 3.6+ | 用于构建后端 |
| Node.js | 16+ | 用于构建前端 |
| MySQL | 5.7+ 或 8.0 | 开发环境数据库 |

### 2.2 数据库初始化

确保MySQL已启动，执行以下命令创建数据库和表：

```bash
mysql -uroot -proot < syswiki-backend/src/main/resources/init.sql
```

该脚本会自动创建 `syswiki` 数据库和所有数据表，并插入一条示例数据。

### 2.3 启动后端服务

```bash
cd syswiki-backend

# 首次构建（下载依赖需要几分钟）
mvn clean package -DskipTests

# 启动服务
java -jar target/syswiki-backend-1.0.0.jar
```

启动成功后，后端服务地址：`http://localhost:8080/syswiki`

如需切换为Oracle数据库，启动时指定prod profile：
```bash
java -jar target/syswiki-backend-1.0.0.jar --spring.profiles.active=prod
```

### 2.4 启动前端服务

```bash
cd syswiki-frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

启动成功后，浏览器访问：`http://localhost:3000`

### 2.5 配置文件说明

**后端配置文件位置：** `syswiki-backend/src/main/resources/`

| 配置项 | 文件 | 说明 |
|--------|------|------|
| 数据库连接 | application-dev.yml | MySQL连接信息（开发环境） |
| 数据库连接 | application-prod.yml | Oracle连接信息（生产环境） |
| AI模型地址 | application.yml | 小米MiMo API地址和Token |
| AI模型名称 | application.yml | flash-model / think-model |

**切换数据库：** 修改 `application.yml` 中 `spring.profiles.active` 为 `dev`(MySQL) 或 `prod`(Oracle)。

**修改AI模型：** 在 `application.yml` 中修改以下配置：
```yaml
syswiki:
  ai:
    base-url: https://token-plan-cn.xiaomimimo.com/v1
    api-key: 你的Token
    flash-model: mimo-v2.5-pro
    think-model: mimo-v2.5-pro
```

## 3. 功能操作指南

### 3.1 系统门户

访问 `http://localhost:3000` 进入系统门户页面。

**查看已有系统：** 页面展示所有已入驻的系统卡片，包含系统名称、代号、负责人和最近更新时间。

**创建新系统空间：**
1. 点击页面右上角「创建系统空间」按钮
2. 填写系统名称（如"超级网上银行"）、系统代号（如"SIB"）、负责人
3. 点击「创建」完成

**进入系统空间：** 点击任意系统卡片，进入该系统的百科空间。

### 3.2 系统简介与技术栈

进入系统空间后默认展示此页面。

- 展示系统核心功能描述和技术栈标签
- 点击右上角「编辑」可跳转到编辑页面修改内容

### 3.3 环境架构

展示测试环境和生产环境的架构信息，通过Tab标签页切换。

### 3.4 服务器与数据库配置

三个子Tab页：
- **基础设施：** 服务器IP、域名、部署模块、CPU/内存配置
- **网络策略：** 防火墙策略、负载均衡配置、域名映射
- **数据库：** 数据库实例状态、连接池配置

### 3.5 快速接入指南

面向上下游开发人员，包含接入流程、Maven依赖、联调注意事项等。

### 3.6 运维SQL与指令库

**查看SQL列表：** 按分类筛选（常用查询、数据校对、应急冲正、性能监控、Shell指令）。

**参数化SQL使用：**
1. 找到需要的SQL条目
2. 在参数输入框中填入值（如交易单号）
3. 点击「生成SQL」，系统自动替换占位符生成完整SQL
4. 点击「复制」按钮一键复制到剪贴板

**示例：**
- 模板：`SELECT * FROM trans_log WHERE bill_no = :billNo`
- 输入参数：`TX20260528001`
- 生成结果：`SELECT * FROM trans_log WHERE bill_no = 'TX20260528001'`

### 3.7 黄金链路拓扑

展示系统的上下游拓扑关系图：
- 节点代表系统/组件，连线代表调用关系
- 支持拖拽、缩放操作
- 点击连线，右侧弹出接口详情面板（协议、接口名称、报文字段说明）

### 3.8 AI智能问答

**快速问答模式（默认）：** 输入问题后，AI基于当前系统知识库回答，响应速度快。

**深度推理模式：** 打开右上角开关，适用于复杂排查场景，如"生产环境网关超时如何排查"。

**使用技巧：**
- 问题越具体，回答越准确
- 可以贴入报错日志让AI分析
- 如果知识库未收录，AI会提示联系系统负责人

### 3.9 内容编辑

**在线编辑：**
1. 在左侧菜单点击「编辑内容」
2. 选择要编辑的模块类型
3. 在Markdown编辑器中修改内容，右侧实时预览
4. 点击「保存」，版本号自动递增

**导入Markdown：**
1. 点击「导入Markdown」按钮
2. 选择本地编写好的 `.md` 文件
3. 系统自动按二级标题拆分并填充到对应模块

**导出Markdown：**
- 点击「导出Markdown」按钮，下载当前模块内容为 `.md` 文件

**Markdown模版下载：**
- 访问 `http://localhost:8080/syswiki/api/templates/markdown-template.md` 下载标准模版

## 5. API接口速查

| 模块 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 空间 | GET | /api/spaces | 空间列表 |
| 空间 | POST | /api/spaces | 创建空间 |
| 内容 | GET | /api/spaces/{id}/contents/{type} | 获取模块内容 |
| 内容 | PUT | /api/spaces/{id}/contents/{type} | 保存内容 |
| 内容 | POST | /api/spaces/{id}/contents/import | 导入Markdown |
| 内容 | GET | /api/spaces/{id}/contents/export | 导出Markdown |
| 拓扑 | GET | /api/spaces/{id}/topologies | 拓扑列表 |
| 拓扑 | POST | /api/spaces/{id}/topologies | 批量保存 |
| SQL库 | GET | /api/spaces/{id}/sql-lib | SQL列表 |
| SQL库 | POST | /api/spaces/{id}/sql-lib/{sqlId}/render | 参数化渲染 |
| AI | POST | /api/spaces/{id}/ai/chat | 智能问答(SSE) |

## 6. 常见问题

**Q: 启动后端报数据库连接失败？**
A: 确认MySQL已启动，用户名密码为root/root，端口3306。检查 `application-dev.yml` 中的连接信息。

**Q: 前端页面空白？**
A: 确认后端服务已启动（端口8080），前端vite.config.ts中的proxy配置指向正确的后端地址。

**Q: AI问答没有响应？**
A: 检查 `application.yml` 中的 `syswiki.ai.base-url` 和 `syswiki.ai.api-key` 是否正确配置。确认网络能访问小米MiMo API。

**Q: 如何切换到Oracle数据库？**
A: 修改 `application.yml` 中 `spring.profiles.active` 为 `prod`，并在 `application-prod.yml` 中配置Oracle连接信息。Oracle表结构需手动执行建表脚本（参考概要设计文档中的DDL）。

**Q: 如何新增系统模块类型？**
A: 1) 在 `ModuleType` 枚举中新增类型；2) 在前端 `constants.ts` 的 `MODULE_TYPE_LABEL` 中添加映射；3) 在对应页面中添加展示逻辑。
