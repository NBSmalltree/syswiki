# 📄 “系统百科”平台需求说明文档（PRD）

## 1. 引言

### 1.1 编写目的

本文档旨在明确“系统百科”平台的建设需求与技术架构。该平台部署于银行内网，旨在打破信息孤岛，解决系统资产分散、跨团队协同难、新员工熟悉系统慢等痛点。平台采用多系统通用的架构设计，首期以某核心系统为标杆进行全量数据填充交付，后续可推广至全行其他系统入驻。

### 1.2 核心目标

1. **结构化资产管理：** 提供统一的 Markdown 模版，实现系统简介、技术栈、服务器、网络、数据库及运维 SQL 的全景结构化管理。
2. **可视化黄金链路：** 基于 ECharts 实现动态拓扑配置，直观展示交易的接口级、报文级全链路走向。
3. **内网 AI 智能问答：** 基于行内 DeepSeek-v4 矩阵，通过 RAG（检索增强生成）技术，提供无幻觉的内网系统知识问答。

---

## 2. 总体架构设计

### 2.1 业务架构拓扑

```
[ 统一门户前端 (Vue3 + ECharts) ]
        |
        v
[ 业务逻辑层 (Java 11 + Spring Boot) ] <---> [ AI 编排层 (LangChain4j) ] <---> [ 行内 DeepSeek API ]
        |                                       |
        v                                       v
[ 关系型存储 (Oracle 11g) ]           [ 向量存储 (轻量级嵌入式/独立向量库) ]

```

### 2.2 平台技术栈（落地版）

* **前端（Frontend）：**
* **核心框架：** Vue 3 + Vite
* **UI 组件库：** Element Plus
* **图表与拓扑：** ECharts 5.x（用于黄金链路与上下游拓扑图的绘制）
* **Markdown 组件：** `v-md-editor` 或 `md-editor-v3`（支持在线编辑与实时预览）


* **后端（Backend）：**
* **开发语言：** Java 11
* **基础框架：** Spring Boot 2.7.x 或 3.x (兼容Java 11版本)
* **AI 整合框架：** LangChain4j（用于对接行内 DeepSeek API、管理 Prompt、处理 RAG 工作流）


* **数据存储（Storage）：**
* **关系型数据库：** Oracle（存储系统元数据、服务器配置、SQL 库、全量 Markdown 文本及版本轨迹）
* **向量数据库：** H2/Derby + Lucene（嵌入式轻量方案） 或 PgVector / Qdrant（视行内软硬件环境而定，由 LangChain4j 统一抽象，用于 RAG 语义检索）

## 3. 功能需求明细

### 3.1 系统空间管理（多系统通用设计）

* **【功能描述】** 平台支持创建多个“系统空间”。首期仅交付一个核心系统的详细百科。
* **【关键特性】**
* 系统列表门户：展示当前已入驻的系统卡片（包含系统名称、系统简称、负责人、最新更新时间）。
* 快速导航：点击卡片进入该系统的专属百科空间。



### 3.2 数据录入与更新（Markdown 驱动）

* **【功能描述】** 平台支持通过“标准化 Markdown 模版”进行离线编写与一键导入，同时支持在线可视化编辑。
* **【关键特性】**
* **模版下载：** 提供统一的系统百科标准 Markdown 模版（包含固定的二级、三级标题层级）。
* **快速导入/导出：** 系统负责人可直接导入写好的 `.md` 文件，系统自动解析标题并填充到对应的页面模块中；同样支持将当前系统的全部或部分配置一键导出为 `.md` 文件进行离线备份或更新。
* **在线编辑：** 集成内网 Markdown 编辑器，支持左边编辑、右边实时预览。
* **轻量版本轨迹：** 底层表结构预留 `version`、`operator`、`update_time` 字段。每次导入或在线保存，版本号自增，记录是谁在什么时间更新了系统配置。



### 3.3 核心百科内容展示模块

#### 3.3.1 系统简介与技术栈

* 展现系统核心功能、支持的业务服务，并以标签或列表形式展示系统当前所使用的技术栈（如 Java、Spring Boot、OceanBase 等）。

#### 3.3.2 测试与生产环境架构

* 展示测试/生产环境的整体物理、逻辑或 XC（信创）架构。通过 Markdown 录入，前端进行多标签页（Tab）切换展示。

#### 3.3.3 服务器、网络与数据库配置

* **基础设施清单：** 包含服务器 IP、内网域名、部署模块、CPU/内存配置。
* **网络策略管理：** 记录**防火墙策略申请状态**（如：已开通、申请中、待调整）、**F5/Nginx 负载均衡配置**（VIP、权重、监听端口）、**域名映射关系**。
* **数据库状态：** 记录数据库实例（如 Oracle、OceanBase）的**主备节点状态**、连接池配置、读写分离策略。

#### 3.3.4 快速接入指南

* 面向上下游开发人员，包含接入流程说明、Maven 依赖配置、测试环境聯調注意事项等。

#### 3.3.5 常见运维 SQL 与指令库

* **分类管理：** 划分为常用查询、数据校对、应急冲正、性能监控等类别。
* **参数化与安全防范：** SQL/指令支持参数占位符（如 `SELECT * FROM trans WHERE bill_no = :billNo`），UI 界面提供输入框，输入单号后**自动生成/替换完整 SQL**，并提供“一键复制”功能。
* **脚本支持：** 包含常用 Linux 运维 Shell 指令（如：特定日志的高效 `grep` 组合、服务重启指令）。

### 3.4 黄金链路与上下游拓扑（ECharts 动态渲染）

* **【功能描述】** 拒绝静态死图，通过在后台或 Markdown 中进行结构化配置（声明节点与连线关系），利用 ECharts 自动渲染出动态拓扑图。
* **【接口/报文级细化】**
* **全局视角：** 展示当前系统处于上下游的什么位置，核心交易（如跨境支付）从哪个系统发起，中途经过哪些中间件/网关，最终在哪个系统落地。
* **微观交互（点击下钻）：** 点击 ECharts 拓扑图上的**连线或系统节点**，下方或侧边栏动态弹出**接口级/报文级定义**：
* 使用的通信协议（如 HTTP/HTTPS、TCP/ISO8583、RPC、MQ）。
* 核心交易码/接口名称。
* 核心请求与响应字段说明。





### 3.5 内网 AI 智能问答（RAG 知识库）

* **【功能描述】** 结合行内 DeepSeek-v4 矩阵，基于当前系统空间内录入的所有百科文本、SQL 库和链路配置进行局域网内的精准 RAG 问答。
* **【关键特性】**
* **无幻觉限缩：** Prompt 严格限制 AI 只能根据当前系统空间的知识库内容回答。若知识库未提及，AI 应礼貌回复“知识库暂未收录该信息，请联系系统负责人”。
* **动态模型调度（策略选型）：**
* **常规事实问答（使用 `deepseek-v4-flash`）：** 如“测试环境核心库的IP是多少？”、“查询交易流水的SQL是什么？”，秒级响应，低延迟。
* **复杂场景推演（提供入口切换至 `think-high/max`）：** 允许用户自主勾选或系统根据复杂查询提示切换。例如用户输入“如果生产环境网关报错超时，应该按什么步骤排查？”，或者贴入长段报错日志时，调用具备深度思考能力（Reasoning）的模型进行链路与配置推演。


* **信息切片与向量化：** 当 Markdown 文件导入或在线更新时，后端通过 LangChain4j 自动触发文本切片（Text Chunking），计算 Embedding 向量并同步更新至向量数据库。

## 4. 非功能需求

1. **安全性（行内合规）：** 平台严禁明文存储生产数据库密码、系统密钥等敏感资产；所有外发至行内 AI 接口的数据需确保在行内局域网内流转，符合行内数据安全规定。
2. **易用性与响应速度：** ECharts 拓扑图节点在 50 个以内时，渲染时间需小于 1 秒；常规 AI 问答（Flash 模型）首字响应（Stream 流式）需在 1.5 秒内。
3. **可扩展性（核心）：** 页面路由与数据库表结构必须解耦“系统 ID”，所有百科页面、拓扑图、AI 向量空间均以 `system_id` 进行隔离，确保新系统入驻时只需“新建系统空间”，无需修改任何底层代码。

## 5. 关键数据库表结构设计（Oracle 示例）

为确保后期平滑支持“版本演进”并契合多系统通用的架构，设计以下核心表结构雏形：

### 5.1 系统空间主表 (`SYS_ENCY_SPACE`)

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `SYSTEM_ID` | VARCHAR2(32) | 主键，系统唯一编号 |
| `SYSTEM_NAME` | VARCHAR2(128) | 系统名称（如：超级网上银行） |
| `SYSTEM_CODE` | VARCHAR2(32) | 系统简称/代号（如：SIB） |
| `OWNER` | VARCHAR2(64) | 系统负责人 |
| `CREATE_TIME` | DATE | 创建时间 |

### 5.2 百科内容明细表 (`SYS_ENCY_CONTENT`)

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `CONTENT_ID` | VARCHAR2(32) | 主键 |
| `SYSTEM_ID` | VARCHAR2(32) | 外键，关联系统主表 |
| `MODULE_TYPE` | VARCHAR2(32) | 模块类型（INTRO-简介/ARCH-架构/SERVER-服务器/SQL-运维SQL等） |
| `MD_CONTENT` | CLOB | Markdown 源码内容 |
| `VERSION` | NUMBER(8) | **版本号（用于平滑演进，每次更新自增）** |
| `OPERATOR` | VARCHAR2(64) | **操作人（记录维护者）** |
| `UPDATE_TIME` | DATE | 更新时间 |

### 5.3 拓扑链路配置表 (`SYS_ENCY_TOPOLOGY`)

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `LINK_ID` | VARCHAR2(32) | 主键 |
| `SYSTEM_ID` | VARCHAR2(32) | 外键，关联系统主表 |
| `FROM_NODE` | VARCHAR2(64) | 起始节点名称（如：核心路由网关） |
| `TO_NODE` | VARCHAR2(64) | 目标节点名称（如：跨境支付通） |
| `PROTOCOL` | VARCHAR2(32) | 通信协议（HTTP/TCP/RPC等） |
| `INTERFACE_DETAILS` | CLOB | 接口与报文详情（Markdown或JSON文本，点击线时展示） |
