# 系统百科平台 — 生产环境部署文档

## 1. 部署架构

```
                        ┌─────────────┐
  浏览器 ──HTTPS──▶    │   Nginx     │
                        │  (端口 443) │
                        └──┬──────┬──┘
                           │      │
                  静态文件  │      │  /api 反向代理
                           ▼      ▼
                  ┌────────┐  ┌──────────────┐
                  │ 前端静态 │  │ Spring Boot  │
                  │ dist/  │  │ (端口 8080)   │
                  └────────┘  └──────┬───────┘
                                     │
                              ┌──────┴───────┐
                              │  Oracle / MySQL │
                              │  (端口 1521/3306)│
                              └────────────────┘
```

## 3. 部署步骤

### 3.1 上传代码到服务器

将整个项目目录上传到服务器，例如：

```bash
scp -r SysWiki/ user@10.x.x.x:/opt/syswiki/
```

最终目录结构：

```
/opt/syswiki/
├── syswiki-backend/          # 后端工程
├── syswiki-frontend/         # 前端工程
└── deploy/                   # 部署产出目录（后续创建）
```

---

### 3.2 数据库初始化

**MySQL 环境：**

```bash
mysql -uroot -p < /opt/syswiki/syswiki-backend/src/main/resources/init.sql
```

**Oracle 环境：**

项目已提供 Oracle 建表脚本 `syswiki-backend/src/main/resources/init_oracle.sql`，直接执行：

```bash
# 先创建表空间和用户（DBA 操作）
sqlplus / as sysdba << EOF
CREATE TABLESPACE syswiki_ts DATAFILE '/data/oracle/syswiki01.dbf' SIZE 500M AUTOEXTEND ON;
CREATE USER syswiki IDENTIFIED BY YOUR_PASSWORD DEFAULT TABLESPACE syswiki_ts;
GRANT CONNECT, RESOURCE, CREATE SESSION TO syswiki;
GRANT UNLIMITED TABLESPACE TO syswiki;
EOF

# 执行建表脚本
sqlplus syswiki/YOUR_PASSWORD@//db-host:1521/orcl @/opt/syswiki/syswiki-backend/src/main/resources/init_oracle.sql
```

**MySQL 与 Oracle 主要差异对照：**

| 项目 | MySQL | Oracle |
|------|-------|--------|
| 大文本 | `LONGTEXT` | `CLOB` |
| 整数 | `INT` | `NUMBER(8)` |
| 时间默认值 | `DEFAULT CURRENT_TIMESTAMP` | `DEFAULT SYSTIMESTAMP` |
| 自增主键 | `AUTO_INCREMENT` | 应用层生成（IdGenerator） |
| 字符集 | `utf8mb4` | 数据库级设置 |
| 引擎 | `ENGINE=InnoDB` | 不需要 |

### 3.3 后端构建与部署

#### 3.3.1 构建 JAR 包

```bash
cd /opt/syswiki/syswiki-backend
mvn clean package -DskipTests
```

产出文件：`target/syswiki-backend-1.0.0.jar`

#### 3.3.2 创建部署目录

```bash
mkdir -p /opt/syswiki/deploy/backend
cp target/syswiki-backend-1.0.0.jar /opt/syswiki/deploy/backend/
```

#### 3.3.3 创建生产环境配置文件

```bash
cat > /opt/syswiki/deploy/backend/application-prod.yml << 'EOF'
server:
  port: 8080
  servlet:
    context-path: /syswiki

spring:
  profiles:
    active: prod
  servlet:
    multipart:
      max-file-size: 20MB
      max-request-size: 20MB
  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: Asia/Shanghai
  datasource:
    # ===== MySQL =====
    url: jdbc:mysql://localhost:3306/syswiki?useUnicode=true&characterEncoding=UTF-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: YOUR_DB_PASSWORD
    driver-class-name: com.mysql.cj.jdbc.Driver
    # ===== Oracle（取消注释切换） =====
    # url: jdbc:oracle:thin:@//db-host:1521/orcl
    # username: syswiki
    # password: YOUR_DB_PASSWORD
    # driver-class-name: oracle.jdbc.OracleDriver

mybatis-plus:
  mapper-locations: classpath:mapper/*.xml
  configuration:
    map-underscore-to-camel-case: true
  global-config:
    db-config:
      id-type: none

syswiki:
  jwt:
    secret: CHANGE_THIS_TO_A_RANDOM_256BIT_KEY_IN_PRODUCTION
    expiration: 86400000
  ai:
    base-url: https://token-plan-cn.xiaomimimo.com/v1
    api-key: YOUR_MIIMO_API_KEY
    flash-model: mimo-v2.5-pro
    think-model: mimo-v2.5-pro

logging:
  level:
    com.syswiki: INFO
    dev.langchain4j: WARN
  file:
    name: /opt/syswiki/logs/backend.log
EOF
```

**需修改的配置项：**

| 配置项 | 说明 |
|--------|------|
| `spring.datasource.password` | 数据库密码 |
| `syswiki.jwt.secret` | JWT 密钥（建议随机生成32位以上字符串） |
| `syswiki.ai.api-key` | 小米 MiMo API Token |
| `syswiki.ai.base-url` | AI 模型地址（内网可达） |

#### 3.3.4 创建启动脚本

```bash
cat > /opt/syswiki/deploy/backend/start.sh << 'EOF'
#!/bin/bash
APP_HOME=/opt/syswiki/deploy/backend
JAR=$APP_HOME/syswiki-backend-1.0.0.jar
LOG_DIR=/opt/syswiki/logs
mkdir -p $LOG_DIR

# JVM 参数
JAVA_OPTS="-Xms512m -Xmx1024m -XX:+UseG1GC"

# 使用外部配置文件
SPRING_OPTS="--spring.config.additional-location=file:$APP_HOME/application-prod.yml"

echo "Starting SysWiki Backend..."
nohup java $JAVA_OPTS -jar $JAR $SPRING_OPTS > $LOG_DIR/startup.log 2>&1 &
echo $! > $APP_HOME/app.pid
echo "PID: $(cat $APP_HOME/app.pid)"
EOF

chmod +x /opt/syswiki/deploy/backend/start.sh
```

#### 3.3.5 创建停止脚本

```bash
cat > /opt/syswiki/deploy/backend/stop.sh << 'EOF'
#!/bin/bash
APP_HOME=/opt/syswiki/deploy/backend
if [ -f $APP_HOME/app.pid ]; then
    PID=$(cat $APP_HOME/app.pid)
    echo "Stopping SysWiki Backend (PID: $PID)..."
    kill $PID
    rm -f $APP_HOME/app.pid
    echo "Stopped."
else
    echo "No PID file found."
fi
EOF

chmod +x /opt/syswiki/deploy/backend/stop.sh
```

#### 3.3.6 启动后端

```bash
cd /opt/syswiki/deploy/backend
./start.sh
```

验证：

```bash
curl http://localhost:8080/syswiki/api/spaces
# 应返回 JSON 响应
```

### 3.4 前端构建与部署

#### 3.4.1 修改 API 地址

编辑 `syswiki-frontend/vite.config.ts`，将 proxy 目标改为生产环境地址：

```typescript
// 生产构建时不需要 proxy，API 由 Nginx 转发
// 保持默认即可，构建时会使用 .env.production 的配置
```

编辑 `syswiki-frontend/.env.production`：

```bash
VITE_API_BASE_URL=/syswiki/api
VITE_APP_TITLE=系统百科
```

#### 3.4.2 构建前端

```bash
cd /opt/syswiki/syswiki-frontend
npm install
npm run build
```

产出目录：`dist/`（包含 index.html + 静态资源）

#### 3.4.3 部署到 Nginx 目录

```bash
mkdir -p /opt/syswiki/deploy/frontend
cp -r dist/* /opt/syswiki/deploy/frontend/
```

### 3.5 Nginx 配置

```bash
cat > /etc/nginx/conf.d/syswiki.conf << 'EOF'
server {
    listen       80;
    server_name  syswiki.internal.bank.com;  # 改为实际域名

    # 前端静态资源
    location / {
        root /opt/syswiki/deploy/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;   # Vue Router History 模式
    }

    # 后端 API 反向代理
    location /syswiki/api/ {
        proxy_pass http://127.0.0.1:8080/syswiki/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE 流式响应支持（AI 问答）
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 120s;

        # 文件上传大小
        client_max_body_size 20m;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        root /opt/syswiki/deploy/frontend;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # 健康检查
    location /health {
        proxy_pass http://127.0.0.1:8080/syswiki/api/spaces;
        access_log off;
    }
}
EOF
```

**验证并重启 Nginx：**

```bash
nginx -t
# 输出 syntax is ok / test is successful 后
systemctl reload nginx
```

## 5. 系统服务化（可选）

将后端注册为 systemd 服务，实现开机自启和自动重启：

```bash
cat > /etc/systemd/system/syswiki.service << 'EOF'
[Unit]
Description=SysWiki Backend Service
After=network.target

[Service]
Type=simple
User=appuser
WorkingDirectory=/opt/syswiki/deploy/backend
ExecStart=/usr/bin/java -Xms512m -Xmx1024m -jar syswiki-backend-1.0.0.jar --spring.config.additional-location=file:./application-prod.yml
ExecStop=/bin/kill -SIGTERM $MAINPID
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable syswiki
systemctl start syswiki
systemctl status syswiki
```

常用命令：

```bash
systemctl start syswiki      # 启动
systemctl stop syswiki       # 停止
systemctl restart syswiki    # 重启
journalctl -u syswiki -f     # 查看日志
```

## 6. 运维注意事项

**日志位置：**
- 后端日志：`/opt/syswiki/logs/backend.log`（或通过 journalctl 查看）
- Nginx 日志：`/var/log/nginx/access.log`、`/var/log/nginx/error.log`

**更新部署流程：**
1. 停止后端服务
2. 替换 JAR 包
3. 重新启动
4. 前端更新：替换 dist 目录文件，Nginx 无需重启

**安全建议：**
- 首次部署后立即修改 admin 默认密码
- JWT secret 必须更换为随机强密钥
- 内网环境建议关闭 Nginx 的 server_tokens
- 数据库密码不要明文写在配置文件中，可用环境变量注入：`DB_PASSWORD=xxx java -jar ...`

**环境变量注入方式：**

```bash
export DB_PASSWORD=your_db_password
export AI_API_KEY=your_api_key
export JWT_SECRET=your_jwt_secret
java -jar syswiki-backend-1.0.0.jar \
  --spring.config.additional-location=file:./application-prod.yml
```

配置文件中使用 `${DB_PASSWORD}` 引用即可。
